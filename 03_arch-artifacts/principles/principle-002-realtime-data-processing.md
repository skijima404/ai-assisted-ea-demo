# Architecture Principle

## Statement
データの連携はリアルタイムで行い、バッチ処理による遅延を排除する。社員が「今」の状況を正確に把握できるよう、即時性を確保する。

## Rationale
現状、バッチ処理による遅延が発生している。社食を利用した翌日にならないと金額が反映されず、新入社員のアカウント有効化が約1週間かかる。また、退職者のアカウント無効化も遅延している。

リアルタイムデータ処理により、給与天引き予定額・利用明細・栄養情報など、社員が「今」の状況を正確に把握できる。また、新入社員のオンボーディング体験が向上し、退職者のセキュリティリスクを迅速に解消できる。

## Implications
- バッチ処理からリアルタイム連携への移行を検討する
- イベントストリーミング、CDCなどで対応できる可能性がある
- 人事システムからのイベント発行を検討する
- リアルタイム性の要件（即座、数分以内、日次など）を明確にする
- データの即時反映により、ユーザー体験を向上させる
