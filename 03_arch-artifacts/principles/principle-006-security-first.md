# Security First

## Statement
セキュリティを最優先とし、退職者のアカウント無効化など、セキュリティリスクを迅速に解消する。認証・認可を適切に管理し、データ保護を確保する。

## Rationale
現状、退職者のアカウントが無効化されず、セキュリティリスクが発生している。また、EOSLバージョンを使用しているため、セキュリティパッチが提供されず、既知の脆弱性に対して無防備な状態が継続している。

セキュリティファーストにより、退職者のアカウント無効化を迅速化し、セキュリティリスクを解消できる。また、継続的にサポートされる技術スタックへの移行により、セキュリティパッチが提供される環境を確保できる。

## Implications
- 退職者のアカウント無効化を自動化する
- 認証・認可システムとの連携を検討する（REST API、イベント連携など。要調査）
- 継続的にサポートされる技術スタックへの移行を検討する
- セキュリティリスクを定期的に評価する
- 監視・検知の仕組みを導入する
