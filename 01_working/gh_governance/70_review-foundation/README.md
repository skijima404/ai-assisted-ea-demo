# Review Philosophy  
アーキテクチャレビューを「続くもの」にし、不可逆リスクを最小化するための思想レイヤーをまとめたフォルダです。  
ここに記述される内容は、チェックリストやCI観点よりも上位の“原理・思想”に属します。

---

## 🎯 Purpose  
Architecture Governance（TOGAF G–H）を実務で成立させるために必要な  
**レビューの第一原理（First Principles）** を整理する。

特に以下を目的とします：

- **不可逆リスクの未然防止**  
  （データ構造破壊、境界侵食、契約の破壊、運用不能化など）
- **現場負荷を増やさないガバナンス**  
  （仕組みとツールで支える／人間の努力に依存しない）
- **リポジトリを正史として扱うための思想整理**
- **EAキャパシティ問題を構造的に解消する**

---

## 📚 What Belongs Here  
このフォルダには、以下の種類の内容を格納します：

### 1. Review Philosophy（レビュー哲学）  
- ガバナンスが失敗する典型パターン  
- “良いレビュー” を規定する前提条件  
- 現場負荷をかけずに継続運用させるための考え方  
- EAのボトルネック構造と、その“仕組みでの解消”の方針  

### 2. Irreversible Risk Model（不可逆リスクモデル）  
- 不可逆リスクカテゴリ（Data / Contract / Boundary / Ops / Security など）  
- raw-notes から抽出されたリスク構造  
- 実案件で頻出するズレの分析（analysisへのリンク）

### 3. Governance Flow（G–Hフェーズの思想）  
- レビュー観点の上位にある「守るべき意図」  
- 観点とCIチェックの境界  
- 人間レビューに残す範囲

---

## 🔧 Relation to Other Folders  
- `60_review_checklist/` … この哲学をもとに作られる“観点実体”  
- `50_principles/` … 組織的なArchitecture Principles  
- `10_raw-notes/` … 現場で起きたズレ（素材）  
- `analysis/` … raw-notes の構造化（思想へ繋がる中間層）

---

## 🧭 How to Update  
1. raw-notesで気づきを書く  
2. analysisで構造化する  
3. 不可逆リスクに該当する思想・原理をここに記述  
4. 必要に応じて review_checklist と CI 観点に反映

---

## ✨ Vision  
レビューは「全文読む作業」ではなく、  
**“不可逆リスクが発生しないことを確認する儀式”** として成立させる。

そのための思想・原理・判断基準を、このフォルダで育てていく。
